# ルール配置と実行保証 — 参照では実行されない

## 発見した知見

エージェント運用ルールは「どのファイルに書くか」で実行保証のレベルが変わる。

| 配置先 | 読まれるタイミング | 実行保証 |
|---|---|---|
| CLAUDE.md / Progress.md（運用ドキュメント） | 毎セッション・タスク開始時に必ず | **高** |
| サブエージェント定義（`.claude/agents/*.md`） | そのエージェントが起動されたときのみ | エージェント自身の動作には高、**起動側への指示には無効** |
| 「詳細は X を参照」のリンク | メインエージェントが読みに行くかはプロンプト次第 | **低（保証なし）** |

具体例（Plan 0020 で発見）: `addf-code-review-agent.md` に「複数ペルソナの集約ルール（メインエージェント向け）」を書いたが、
このファイルを読むのは**起動されたサブエージェント**であり、集約を実行すべき**メインエージェント**は読む保証がなかった。
運用ドキュメント側に「参照せよ」と書いても、参照は省略されうる。

## プロジェクトへの適用

- **実行してほしい主体が必ず読むファイルに、実行してほしい手順を書く**
- サブエージェント定義に書いてよいのは「そのエージェント自身の振る舞い」だけ
- 起動側（メインエージェント）への指示は、運用ドキュメント（ProgressTemplate / CLAUDE.md）に書く
- 手順が長くて参照に逃げたい場合は、**コア手順（3行以内）をインライン展開し、詳細だけ参照に出す**:
  - 例: 「集約: 同一指摘は1件にまとめペルソナを列挙。2ペルソナ以上の独立指摘は重要度+1。詳細は agent 定義を参照」

## 応用: セッション状態の保存先としての CLAUDE.local.md

「毎セッション参照すべき状態」（例: `/addf-mode` の3軸モード設定）は `CLAUDE.local.md` に書くと、
毎セッション自動読込されるため**新しいブートステップを追加せずに**実行保証される。

- `.gitignore` 済みのため開発者ごとに独立した設定になる（コミット汚染なし）
- 専用の状態ファイル（`.claude/.addf-mode` 等）+ ブートステップ追加よりも、読み込み保証・実装コストの両面で優る
- 書き込み時は対象セクション（例: `# ADDF モード`）だけを更新し、ユーザーの他の記述に触れないこと

## 注意点・制約

- インライン展開はテンプレート肥大化とのトレードオフ。「実行に必須の判断基準」だけを展開し、背景・理由は参照先に残す
- `.addf.md` テンプレートと運用中ファイル（Progress.md）の両方に展開する場合、同期忘れに注意（Feedback.md の改善アクション参照）
