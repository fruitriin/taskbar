# Plan: 同一アプリのウィンドウグルーピングと並び順制御

**実装状況**: 実装済み（2026-07-05。D&D の実機確認とリリース時の version 2.1.1+ 更新が残）
**作成日**: 2026-07-05

## 目的

タスクバー上のウィンドウボタンを同じアプリでまとめて表示し、並び順を予測可能・制御可能にする。

1. **同じアプリでまとめる** — 同一 `kCGWindowOwnerName` のウィンドウを隣接させてグループ表示する
2. **アプリ内の並び順** — グループ内の各ウィンドウは「起動順」または「座標順」で並ぶ（ユーザーが選択）
3. **アプリグループ自体の並び順** — ドラッグアンドドロップで並び替えられるのが理想

## 現状の実装

- [index.vue](../../src/renderer/src/pages/index.vue) の `sort()`（187行目付近）は **headers / footers エリア専用**。
  `kCGWindowOwnerName` 昇順 → `kCGWindowBounds.X` 昇順/降順でソートしている
- **center エリア（大半のウィンドウ）はソートされておらず、Helper の送信順のまま**。
  Helper の更新タイミングで並びが揺れるのが体感上の問題の根源
- [store.ts](../../src/main/funcs/store.ts) の `options` に `windowSortByPositionInApp: false` が既に定義済みだが、**現在どこからも参照されていない**（先行して置かれた設定項目）。本計画で UI とロジックを接続する

## 設計方針

### 1. グルーピング

- center エリアに対して「アプリ名でグループ化 → グループを平坦化して描画」の computed を追加する
- グループの既定順序: タスクバーに最初に出現した順（セッション内で安定）。
  新しいアプリは末尾に追加される
- headers / footers は現行仕様（設定画面の並び順に従う）を維持する

### 2. アプリ内のウィンドウ並び順（オプション切替）

| モード | ソートキー | 備考 |
|---|---|---|
| 起動順（デフォルト） | `kCGWindowNumber` 昇順 | ウィンドウ生成順に単調増加するため起動順の近似として使える |
| 座標順 | `kCGWindowBounds.X` 昇順（縦レイアウト時は `Y` 昇順） | 画面上の位置と一致 |

- 既存の `options.windowSortByPositionInApp`（boolean）をそのまま使う: `false` = 起動順、`true` = 座標順
- 設定画面（option.vue）にトグルを追加し、`setOptions` IPC で保存 → `updateOptions` で全タスクバーに反映

### 3. アプリグループの D&D 並び替え

- `options.appOrder: string[]`（アプリ名の配列）を electron-store に新設し、グループ順序の上書きに使う
  - `appOrder` に載っていないアプリは出現順で末尾に続く
  - マイグレーション追加: `>=2.1.1` で `appOrder: []` を補う
  - **注意**: electron-store のマイグレーションは `semver.satisfies(アプリの version, キー)` で発火判定される。
    現行バージョンは 2.1.0 のため、**リリース時に package.json の version を 2.1.1 以上へ上げること**。
    それまでの間もレンダラー側の `?? []` フォールバックと D&D 時の明示書き込みで自己修復される
- タスクバー上でウィンドウボタン（またはグループ単位のハンドル）をドラッグして並び替え、ドロップ時に `setOptions` で永続化する
- D&D 実装は依存を増やさず HTML5 Drag and Drop API を第一候補とする（`vuedraggable` は Vue 3 対応版の保守状況を確認してから判断）

## 実装ステップ

1. **ソート純関数の切り出し**: グルーピング＋アプリ内ソートを `utils.ts` に純関数として実装し、bun test でユニットテストを書く（`kCGWindowNumber` / 座標 / appOrder の組み合わせ）
2. **center エリアへの適用**: index.vue の `centerWindows` に純関数を接続。ブラウザモック（`__mockHelpers.updateWindows`）で並びの安定性を確認
3. **設定 UI**: option.vue に「アプリ内の並び順（起動順/座標順）」トグルを追加
4. **D&D 並び替え**: タスクバー上のグループ D&D → `appOrder` 永続化。Playwright MCP で操作テスト
5. **リリース確認**: `mise run test:all` / `typecheck` / 実機で Helper 込み動作確認

ステップ 1〜3 だけでも「並びが揺れない・アプリでまとまる」という主要価値が出るため、D&D（ステップ4）は独立してリリース可能。

## 未決事項

- グループ内ウィンドウの D&D 並び替え（アプリ間だけでなくウィンドウ単位）まで対応するか → 初版はアプリグループ単位のみ
- マルチディスプレイ時に `appOrder` をディスプレイごとに持つか共通にするか → 初版は共通
- グループの視覚表現（区切り・アイコン集約表示など）を入れるか → 初版は並び順のみ（見た目は現状踏襲）
