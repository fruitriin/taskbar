# 経験ファイル メンション検証

スキル定義内の `.exp.md` への参照（@メンション）が正しく展開される書式になっているかを検証し、必要に応じて修正する。

## 背景

Claude Code の @メンション展開では、クオート（バッククォート）で囲まれた @FileName は展開されずリテラル扱いになる。
経験ファイル（`.exp.md`）はスキル実行時に自動展開されるべきなので、クオートで囲まれていてはならない。

## 引数
- **引数なし**: 全スキルを検証する
- `$ARGUMENTS`: 特定のスキル名（例: `addf-gui-test`）を指定して検証する

## 手順

### Phase 1: スキャン

1. `.claude/commands/` と `.claude/optional/` 配下の全 `.md` ファイルを一覧取得する（サブディレクトリ含む。optional はオプトイン式スキルの原本置き場）
2. 各ファイルの内容から `.exp.md` を含む行を抽出する
3. 以下のパターンを分類する:
   - **正常**: `addf-foo.exp.md`（クオートなし、@展開される）
   - **要修正**: `` `addf-foo.exp.md` `` や ` ```...addf-foo.exp.md...``` `（クオート内、展開されない）
   - **例外**: コードブロック内やコメントで意図的にリテラル表記しているもの

### Phase 2: 判定

4. 各検出箇所について修正が必要か判断する:
   - 「経験の活用」セクション内の `.exp.md` 参照 → **展開すべき（クオート不要）**
   - 手順説明内でファイル名を言及するだけの箇所 → **リテラルで正しい（クオート維持）**
   - コードブロック内の例示 → **リテラルで正しい（変更不要）**

### Phase 3: 修正

5. 修正が必要な箇所のクオートを外す
6. 修正内容を一覧で報告する:
   ```
   ## 検証結果

   | ファイル | 行 | 修正前 | 修正後 | 状態 |
   |---|---|---|---|---|
   | addf-gui-test.md | 37 | `addf-gui-test.exp.md` | addf-gui-test.exp.md | 修正済 |
   | addf-knowhow.md | 46 | — | — | 正常 |
   ```

### Phase 4: 検証

7. 修正後のファイルを再スキャンし、意図しない変更がないことを確認する
8. 修正がなかった場合は「全スキルの .exp.md 参照は正常です」と報告する

## エラーケース

- `.claude/commands/` が存在しない → 「commands ディレクトリが見つかりません」と報告
- `.exp.md` 参照が1つもないスキル → スキップ（経験活用セクションがないスキルもある）
- @メンションのパスが実在しないファイルを指している → 警告として報告（exp ファイルはスキル実行後に初めて作成されるため、未存在は正常）

## 経験の活用
- 実行前に `addf-experience.exp.md` が存在すれば読み、過去の経験を考慮する
- 実行後、新たな教訓があれば `addf-experience.exp.md` に追記する
